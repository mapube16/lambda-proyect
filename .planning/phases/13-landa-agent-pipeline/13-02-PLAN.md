**Add post-scoring routing** — after scoring produces puntaje, add routing logic:
```python
from landa.state_machine import update_lead_estado
if lead_id and user_id:
    if puntaje < 40:
        # Mark as rejected by AI — no estado transition
        await db.leads.update_one(
            {"_id": ObjectId(lead_id)},
            {"$set": {"system_state": "REJECTED_BY_AI"}}
        )
    elif 40 <= puntaje < 70:
        await db.leads.update_one(
            {"_id": ObjectId(lead_id)},
            {"$set": {"motivo_nurturing": "score_bajo"}}
        )
        await update_lead_estado(lead_id, user_id, "nurturing")
    else:  # >= 70
        await update_lead_estado(lead_id, user_id, "checkpoint")
```

Note: The routing block must be wrapped in try/except — if update_lead_estado raises ValueError (invalid transition from current estado), log the error and continue. Leads may not have estado="investigando" if already processed.
