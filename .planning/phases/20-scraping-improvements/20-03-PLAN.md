**In prospector.py — replace the outer client block in scrape_url():**

Replace the entire block from `html = ""` through `last_error = str(e)` (shown in interfaces above) with the curl_cffi version. The ua_profiles list (lines 888–911) and the build_candidates() inner function (lines 913–945) stay UNCHANGED — only the HTTP client block changes.

New outer client block:
```python
    html = ""
    last_error = "no response"
    candidates = build_candidates(url)
    _NO_RETRY_CODES = {400, 401, 403, 404, 405, 410, 429, 451, 500, 502, 503, 504}
    _HARD_BLOCK_CODES = {403, 429, 451}
    try:
        async with AsyncSession(
            impersonate="chrome131",
            allow_redirects=True,
            timeout=timeout,
        ) as client:
            hard_blocked = False
            for candidate in candidates:
                if hard_blocked:
                    break
                try:
                    resp = await client.get(candidate)
                except RequestsError as e:
                    last_error = f"{candidate}: {e}"
                    continue

                if resp.status_code >= 400:
                    last_error = f"{candidate}: HTTP {resp.status_code}"
                    if resp.status_code in _HARD_BLOCK_CODES:
                        hard_blocked = True
                    if resp.status_code in _NO_RETRY_CODES:
                        break
                    continue

                html = resp.text or ""
                if html.strip():
                    break
    except Exception as e:
        last_error = str(e)
```

Key differences from the old block:
- `httpx.AsyncClient(follow_redirects=True, timeout=timeout)` → `AsyncSession(impersonate="chrome131", allow_redirects=True, timeout=timeout)`
- Inner `for headers in ua_profiles:` loop is REMOVED — curl_cffi generates authentic Chrome131 headers automatically; manually overriding them would degrade the TLS fingerprint
- `except Exception` in inner loop → `except RequestsError` — more specific; outer `except Exception` still catches connection-level failures
- `client.get(candidate, headers=headers)` → `client.get(candidate)` — no headers argument

Do NOT delete the ua_profiles list — it is defined but no longer used inside scrape_url(). Leave it in place to avoid a potentially disruptive diff. The executor may optionally add a `# ua_profiles kept for reference — no longer used with curl_cffi` comment.
