```python
        # Guard: override generic industria values the LLM sometimes passes instead of the real industry
        _GENERIC_INDUSTRIA = {"empresas", "empresa", "negocios", "companies", "company", "business", ""}
        campaign_industria = (campaign.get("industria_objetivo") or "").strip()
        if campaign_industria and industria.strip().lower() in _GENERIC_INDUSTRIA:
            logger.warning(
                "[discover_companies] LLM passed generic industria=%r; overriding with campaign industria_objetivo=%r",
                industria, campaign_industria,
            )
            industria = campaign_industria
```

Notes:
- `campaign` is already in scope via closure (captured by make_prospecting_registry).
- `_GENERIC_INDUSTRIA` is defined locally inside the guard block, not at module level — it only exists at the point of the guard check.
- The guard runs ONLY when `campaign_industria` is non-empty AND the LLM's industria is a known generic word. If `campaign_industria` is empty (no campaign), the guard is a no-op.
- A valid industry like `"Logística y Transporte"` is NOT in `_GENERIC_INDUSTRIA` and passes through unchanged.
