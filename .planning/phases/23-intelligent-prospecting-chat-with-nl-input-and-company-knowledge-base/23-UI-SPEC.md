# Phase 23 — UI Design Contract: Intelligent Prospecting Chat

> Visual and interaction contract for the NL chat input, extracted-params confirmation card, company knowledge base panel, and lead feedback learning indicator.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

---

## Scope

Phase 23 modifies one existing component and adds one new sub-component to `AgentPanel.tsx`:

1. **NL Chat Input** — Replaces the existing `CampaignChat` start-screen call-to-action ("Configura tu campaña con IA / Comenzar →") with a single-line text input that accepts a free-form NL description. The existing multi-turn `CampaignChat` UI (bubbles, input row, send button) remains as the fallback rendering path when the backend returns `status: "needs_clarification"`. The NL single-turn path is new.

2. **Extracted Params Confirmation Card** — After a successful single-turn NL extraction (`status: "extracted"`), renders a read-only card showing the extracted campaign parameters before the user launches. A "Editar manualmente" toggle expands the existing campaign form grid beneath it (already present in the component).

3. **Company Knowledge Base Panel** — A collapsible section inside the "Campaña" tab, placed below the confirmation card and above the launch button. The tenant can describe their product and ICP in a textarea. Saved on blur to `POST /api/knowledge`. Not a modal — inline panel.

4. **Learning Indicator** — A passive text badge displayed inside the knowledge base panel. Shows a count of approved and rejected lead signals that have been accumulated. No interactive elements.

## Component Inventory

### 1. NL Prospect Input (new sub-component: `NLProspectInput`)

**Layout:** replaces the `CampaignChat` "not started" screen. Full-width within `s.tabContent` padding (14px).

**Idle state:**
- Single `<textarea>` (3 rows, resizable: none) with placeholder text (see copywriting)
- Send button (`➤`) to the right — same gradient as existing `cc.sendBtn`
- Below the textarea: one line of hint text at 11px / `rgba(227,224,241,0.35)` / Space Grotesk

**Loading state (in-flight):**
- Textarea `disabled`, opacity 0.6
- Send button replaced with `"..."` text, `disabled`, opacity 0.5
- No spinner — matches existing loading pattern in `CampaignChat` (`loading && <div style={{ ...cc.bubble, ...cc.bubbleAI, color: '#888' }}>...</div>`)

**Clarification state (`status: "needs_clarification"`):**
- Renders the existing full `CampaignChat` bubble list below the textarea
- The assistant reply message appears as a `cc.bubbleAI` bubble
- Input row remains active for follow-up messages
- This reuses `CampaignChat` exactly — no new component needed

**Error state:**
- Single error message renders as `cc.bubbleAI` bubble with `color: '#ff6188'`
- No modal, no toast — inline within the chat area

### 2. Extracted Params Confirmation Card (inline in "Campaña" tab)

**Replaces:** the existing `paramGrid` + edit toggle — shown only after successful NL extraction

**Structure:**
- Header row: label "Parámetros extraídos" at 9px/600/uppercase/`#ab9df2` (matches `s.sectionTitle`) + count badge showing number of filled fields at 9px/600/`#78dce8`
- Grid: `display: grid; gridTemplateColumns: 1fr 1fr; gap: 8px` — identical to existing `s.paramGrid`
- Each cell: `s.paramCard` style (background `#12121d`, borderRadius 8px, padding `10px 12px`, border `1px solid rgba(120,220,232,0.05)`)
- Field label: 9px / 600 / uppercase / `rgba(227,224,241,0.35)` — `s.paramLabel`
- Field value: 14px / 500 / `#e3e0f1` — `s.paramValue`, truncated with `textOverflow: ellipsis`
- Missing/inferred fields: value displayed as `"—"` in `rgba(227,224,241,0.25)`

**Edit toggle (existing behavior preserved):**
- Button: "✎ Editar parámetros" at 11px / `rgba(227,224,241,0.35)` — `s.toggleBtn`
- Expands the existing `s.campaignForm` grid beneath the confirmation card
- When form is open: button reads "↑ Ocultar"

**Interaction:**
- Confirmation card appears immediately after API response — no transition animation
- The form grid is hidden by default (`showForm: false`) — user must explicitly expand it
- Both the confirmation card and the edit form are present in DOM simultaneously when form is open (card above, form below)

### 3. Company Knowledge Base Panel (inline in "Campaña" tab)

**Position:** below the extracted params confirmation card / above the max prospects slider section.

**Collapsed state (default):**
- Header row: label "Base de conocimiento" at 9px/600/uppercase/`#ab9df2` + toggle button "▼ Expandir" at 11px / `rgba(227,224,241,0.35)`
- Entire panel background: `#12121d`, borderRadius 10px, padding `12px 14px` — matches `s.beePreview` / `s.sliderSection`
- Learning indicator badge visible even in collapsed state (see §4 below)

**Expanded state:**
- Product description textarea: 4 rows, width 100%, background `#1b1a26`, border none, borderBottom `1px solid rgba(120,220,232,0.2)` idle / `rgba(120,220,232,0.4)` focus, color `#e3e0f1`, fontSize 13px, fontFamily Space Grotesk, borderRadius 6px, padding `9px 13px`, resize: vertical, outline none
- Label above textarea: "Tu producto / ICP" at 9px/600/uppercase/`rgba(227,224,241,0.4)` — matches `s.fieldLabel`
- Save behavior: auto-save on blur — no explicit save button
- Save confirmation: field label temporarily changes to "Guardado ✓" at `#a9dc76` for 1.5 seconds then reverts
- Toggle button changes to "↑ Ocultar" when expanded

**Interaction rules:**
- Panel is collapsed by default on first render
- Panel remembers expanded/collapsed state within the session (local React state, not persisted)
- Textarea content is fetched on mount via `GET /api/knowledge` and pre-filled if a document exists
- Auto-save fires on `onBlur` — not on every keystroke — consistent with existing field patterns

### 4. Learning Indicator Badge

**Position:** inside the knowledge base panel header row, between the "Base de conocimiento" label and the expand toggle.

**Visual:**
- Inline span: `fontSize: 10px`, `fontFamily: Space Grotesk`, `color: rgba(227,224,241,0.4)`, `border: 1px solid rgba(120,220,232,0.15)`, `borderRadius: 4px`, `padding: '2px 7px'`, background `rgba(120,220,232,0.04)`
- Copy when signals exist: `"N aprobados · M rechazados"` where N and M are counts from `prospecting_knowledge` document
- Copy when no signals yet: `"Sin señales aún"` at the same styles
- No icon — text only
- Not clickable — purely informational

**Data source:** fetched alongside knowledge base content from `GET /api/knowledge` response — fields `approved_lead_signals.length` and `rejected_lead_signals.length`.

## Interaction Contracts

### NL Submission Flow

1. User types NL description in textarea (min 1 char to enable send)
2. User presses Enter or clicks send button (`➤`)
   - Shift+Enter inserts newline (does NOT submit)
3. Loading state activates immediately — textarea disabled, send button shows `"..."`
4. API call: `POST /api/chat/prospect` with `{ message: <text> }`
5. Response routing:
   - `status: "extracted"` → hide NL input, show extracted params confirmation card → `setCampaignReady(true)` path
   - `status: "needs_clarification"` → show the assistant reply as a `bubbleAI` div, keep input active for follow-up
   - Network error → show error bubble (`color: '#ff6188'`) inline, re-enable input

### Confirmation Card → Launch

After NL extraction, the existing campaign flow resumes unchanged:
1. Confirmation card visible + edit toggle available
2. Max prospects slider visible
3. "INICIAR PROSPECCIÓN" button visible
4. "↩ Nueva campaña" button visible (resets to NL input, clears confirmation)

### Knowledge Base Save

- Fetch on mount: `GET /api/knowledge` → pre-fill textarea
- Auto-save: `onBlur` → `POST /api/knowledge` with `{ product_description: <text> }`
- Visual feedback: label text changes to "Guardado ✓" (`color: '#a9dc76'`) for 1500ms then resets
- Error on save: label text changes to "Error al guardar" (`color: '#ff6188'`) for 2000ms then resets — no modal

### Double-Submit Guard

The send button is `disabled` while `loading === true`. React `useState` pattern — no debounce needed. Consistent with existing `CampaignChat` send guard: `disabled={loading || !input.trim()}`.
