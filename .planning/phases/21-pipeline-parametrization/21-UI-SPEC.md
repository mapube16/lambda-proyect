# Phase 21 — UI Design Contract: Pipeline Parametrization

> Visual and interaction contract for the signal source selector and pipeline configuration UI.
> Generated by gsd-ui-researcher. Revised to sector-agnostic framing. Consumed by gsd-planner and gsd-executor.

---

## Scope

This phase adds one new interaction surface to the existing `AgentPanel.tsx` campaign configuration tab:

**Signal Source Selector** — a set of toggleable cards placed between the "Configurar Campaña" header and the existing `CampaignChat` component. The user picks one or more signal sources before starting the campaign. The selection:
1. Activates the correct discovery flags (`use_rues`, `use_fincaraiz`, `use_secop`) in the campaign state
2. Works for any sector — not tied to insurance or any specific vertical
3. Persists with the campaign document

Additionally, the `FuentesPanel` in `StaffDashboard.tsx` gains two new checkbox rows for the new signal sources (RUES, Fincaraíz) following the exact existing pattern.

## Component Inventory

### 1. SignalSourceSelector (NEW — inside AgentPanel campaign tab)

**Placement:** Inserted between the `campaignHeader` section and the `CampaignChat` component, visible when `!campaignReady`. Also visible in read-only mode above the `paramGrid` when `campaignReady === true`.

**Structure:**
```
<div style={signalGrid}>           // 2-column grid, gap 8px
  <SignalCard source="serper" />       // always enabled, non-toggleable
  <SignalCard source="rues" />
  <SignalCard source="secop" />
  <SignalCard source="fincaraiz" />
</div>
```

**SignalCard states:**

| State | Border | Background | Shadow |
|-------|--------|------------|--------|
| Default | 2px solid transparent | `#1b1a26` | none |
| Hover | 2px solid rgba(120,220,232,0.25) | `#22212e` | none |
| Selected | 2px solid #78dce8 | `#1b1a26` | `0 0 0 1px rgba(120,220,232,0.12)` |
| Always-on (Serper) | 2px solid rgba(120,220,232,0.4) | `#1b1a26` | none, lock icon shown |
| Disabled | 2px solid transparent | `#12121d` | none, opacity 0.4 |

**SignalCard anatomy (top to bottom):**
1. Icon: 24px emoji, `lineHeight: 1`, `marginBottom: 4px`
2. Title: 14px, weight 600, Space Grotesk, `#e3e0f1`
3. Description: 13px, weight 400, Inter, `rgba(227,224,241,0.5)`, `lineHeight: 1.5`
4. Source tag row: horizontal flex, `gap: 4px`, `flexWrap: wrap`, `marginTop: 8px`

**Signal source tag:** `fontSize: 10px`, `fontFamily: Space Grotesk`, `fontWeight: 600`, `textTransform: uppercase`, `letterSpacing: 0.06em`, `padding: '2px 6px'`, `borderRadius: 4px`. Colors from Color section above.

**Signal source definitions:**

| Source | Icon | Title | Description | Campaign flag | Always on |
|--------|------|-------|-------------|---------------|-----------|
| serper | 🌐 | Búsqueda Web | Google + Bing: empresas en cualquier sector y ciudad | — (base layer) | ✓ |
| rues | 🏢 | Empresas recién creadas | Compañías registradas recientemente en Cámara de Comercio | `use_rues: true` | — |
| secop | 📋 | Contratistas del Estado | Proveedores con contratos públicos adjudicados (SECOP) | `use_secop: true` | — |
| fincaraiz | 🏠 | Propietarios con arriendo | Inmuebles en arriendo activo en Fincaraíz | `use_fincaraiz: true` | — |

**Grid layout:** `gridTemplateColumns: '1fr 1fr'`, `gap: 8px`. Multi-select: any combination of rues/secop/fincaraiz can be active simultaneously. Serper is always on and shown as permanently selected.

**Selection indicator:** Full border + subtle shadow only (no left accent bar).

## Interaction Contract

### Selection flow (AgentPanel)

1. User opens "Campaña" tab — signal selector renders above CampaignChat
2. Serper card renders in always-on state (no toggle, lock icon visible)
3. User clicks rues/secop/fincaraiz cards — each toggles independently (multi-select)
4. Selected sources set their respective flags in campaign state (`use_rues`, `use_secop`, `use_fincaraiz`)
5. Any combination is valid — zero optional sources selected is fine (Serper alone runs)
6. Signal selection does NOT auto-start the CampaignChat — user still clicks "Comenzar →"
7. After `campaignReady === true`, the SignalBadge renders in the paramGrid (read-only)
8. Clicking "↩ Nueva campaña" resets signal source selections along with all other campaign state

### Keyboard / accessibility

- Signal cards are `<button>` elements (not `<div>`) for keyboard focus
- `aria-pressed="true"` on selected cards, `aria-disabled="true"` on always-on Serper card
- `aria-label="Señal [título]: [descripción]"` on each card button
- Focus ring: `outline: 2px solid #7c3aed` at `outline-offset: 2px` (existing pattern from index.css)

### Campaign flags set per source

| Flag | Set when |
|------|----------|
| `use_rues` | RUES card selected |
| `use_secop` | SECOP card selected |
| `use_fincaraiz` | Fincaraíz card selected |

All flags default to `false`. Serper (búsqueda web) runs unconditionally — no flag needed.
