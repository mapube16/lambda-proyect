# Phase 21 — UI Design Contract: Pipeline Parametrization

> Visual and interaction contract for the vertical selector and signal source configuration UI.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

---

## Scope

This phase adds one new interaction surface to the existing `AgentPanel.tsx` campaign configuration tab:

**Vertical Selector** — a set of clickable cards placed between the "Configurar Campaña" header and the existing `CampaignChat` component. The user picks an insurance vertical before or during campaign setup. The selection:
1. Pre-fills campaign fields via a vertical → defaults mapping
2. Activates the correct signal sources (use_rues, use_fincaraiz, use_secop flags)
3. Persists with the campaign document

Additionally, the `FuentesPanel` in `StaffDashboard.tsx` gains two new checkbox rows for the new signal sources (RUES, Fincaraíz) following the exact existing pattern.

## Component Inventory

### 1. VerticalSelectorGrid (NEW — inside AgentPanel campaign tab)

**Placement:** Inserted between the `campaignHeader` section and the `CampaignChat` component, visible when `!campaignReady`. Also visible in read-only mode above the `paramGrid` when `campaignReady === true`.

**Structure:**
```
<div style={verticalGrid}>           // 2-column grid, gap 8px
  <VerticalCard vertical="arrendamiento" />
  <VerticalCard vertical="desempleo" />
  <VerticalCard vertical="empresarial" />
  <VerticalCard vertical="personalizado" />   // full width, grid-column: 1/-1
</div>
```

**VerticalCard states:**

| State | Border | Background | Shadow |
|-------|--------|------------|--------|
| Default | 2px solid transparent | `#1b1a26` | none |
| Hover | 2px solid rgba(120,220,232,0.25) | `#22212e` | none |
| Selected | 2px solid #78dce8 | `#1b1a26` | `0 0 0 1px rgba(120,220,232,0.12)` |
| Disabled | 2px solid transparent | `#12121d` | none, opacity 0.4 |

**VerticalCard anatomy (top to bottom):**
1. Icon: 24px emoji, `lineHeight: 1`, `marginBottom: 4px`
2. Title: 14px, weight 600, Space Grotesk, `#e3e0f1`
3. Description: 12px, weight 400, Inter, `rgba(227,224,241,0.5)`, `lineHeight: 1.5`
4. Signal source tags row: horizontal flex, `gap: 4px`, `flexWrap: wrap`, `marginTop: 8px`

**Signal source tag:** `fontSize: 10px`, `fontFamily: Space Grotesk`, `fontWeight: 600`, `textTransform: uppercase`, `letterSpacing: 0.06em`, `padding: '2px 6px'`, `borderRadius: 4px`. Colors from Color section above.

**Vertical definitions:**

| Vertical | Icon | Title | Description | Signal Sources |
|----------|------|-------|-------------|----------------|
| arrendamiento | 🏠 | Pólizas de Arrendamiento | Propietarios con inmuebles en arriendo activo | Serper + Fincaraíz |
| desempleo | 👷 | Pólizas de Desempleo | Empresas que contratan personal nuevo (RUES) | Serper + RUES |
| empresarial | 🏛️ | Pólizas Empresariales | Empresas con contratos públicos adjudicados | Serper + SECOP |
| personalizado | ⚙️ | Configuración Personalizada | Configura las fuentes manualmente | Serper (base) |

**Grid layout:** `gridTemplateColumns: '1fr 1fr'`, `gap: 8px`. "Personalizado" card spans full width (`gridColumn: '1 / -1'`).

**Selected state indicator:** Left border accent bar is NOT used (reserved for FuentesPanel). Selection communicated via full border + subtle shadow only.

## Interaction Contract

### Selection flow (AgentPanel)

1. User opens "Campaña" tab — vertical selector renders above CampaignChat
2. User clicks a vertical card — card enters selected state immediately (optimistic, no loading)
3. Campaign defaults for that vertical are merged into the `campaign` state object (pre-filling fields)
4. Signal source flags (`use_rues`, `use_fincaraiz`, `use_secop`) are set in campaign state
5. If user selects "Personalizado": no pre-fill, all sources default to off, CampaignChat starts from scratch
6. Vertical selection does NOT auto-start the CampaignChat — user still clicks "Comenzar →"
7. After `campaignReady === true`, the VerticalBadge renders in the paramGrid (read-only)
8. Clicking "↩ Nueva campaña" resets vertical selection along with all other campaign state

### Keyboard / accessibility

- Vertical cards are `<button>` elements (not `<div>`) for keyboard focus
- `aria-pressed="true"` on selected card
- `aria-label="Vertical [título]: [descripción]"` on each card button
- Focus ring: `outline: 2px solid #7c3aed` at `outline-offset: 2px` (existing pattern from index.css)

### Campaign defaults per vertical

These are pre-fill values injected into the campaign state when a vertical is selected. The user can override via the edit form.

| Field | arrendamiento | desempleo | empresarial |
|-------|---------------|-----------|-------------|
| industria_objetivo | Sector inmobiliario y arrendamiento | Sector empresarial y RRHH | Sector público y contratistas |
| dolor_operativo | Riesgo de impago de arrendamiento | Riesgo de cesantía del trabajador | Riesgo de incumplimiento de contratos |
| solucion_ofrecida | Póliza de arrendamiento que protege al propietario | Póliza de desempleo para empleados nuevos | Póliza empresarial para contratistas SECOP |
| signal flags | use_fincaraiz: true | use_rues: true | use_secop: true |

`personalizado` vertical: no pre-fill, all flags false, user configures manually.
