<task type="auto" tdd="true">
  <name>Task 3: run_sync orchestrator</name>
  <files>backend/softseguros/sync.py</files>
  <behavior>
    async def run_sync(db, user_id: str, mode: Literal["onboarding","cron_daily","manual"]) -> dict
    - Reads credentials via softseguros.credentials.get_credentials(db, user_id) — raises NoCredentialsError if missing
    - Creates SoftSegurosAdapter, calls authenticate()
    - Creates softseguros_sync_logs entry with started_at, mode, status=in_progress
    - Uses asyncio.Semaphore(5) for all SOFTSEGUROS calls
    - Phase A — paginated fetch:
      * Loop adapter.list_pagopoliza(page=N) until response next is null OR results empty
      * Filter comisionada=false at the application level (in case API doesn't filter)
      * Filter fecha_pago <= today + 30 (skip "futuro")
    - Phase B — enrichment (with in-memory cache):
      * For each pagopoliza: resolve poliza_id → poliza_cache → cliente_id → cliente_cache
      * Use asyncio.gather under the semaphore
    - Phase C — upsert:
      * For each enriched item: classify_pagopoliza, then upsert_debtor_by_softseguros_id
      * Track seen_pagopoliza_ids and increment counters (created vs updated by comparing upsert result.upserted_id)
    - Phase D — soft-delete missing (skip on mode="onboarding" since no prior state):
      * existing_ids = list_active_softseguros_pagopoliza_ids(db, user_id)
      * missing = existing_ids - seen_pagopoliza_ids
      * For each missing id: GET /api/pagopoliza/{id}:
        - If 200 and comisionada=true → mark_debtor_paid_by_softseguros_id, increment debtors_marked_paid
        - If 404 → mark_debtor_deleted_by_softseguros_id, increment debtors_marked_deleted
        - Other errors → log warning, leave is_active unchanged
    - Phase E — finalize:
      * Update softseguros_sync_logs entry with completed_at, status='success', counters, total_requests, duration_seconds
      * On exception: status='failed', error_message=str(e), still set completed_at and persist
      * await adapter.close()
    - Returns the sync_log dict
  </behavior>

